From Industrial RAG Demo to Agentic RCA

A read-only NeMo Agent Toolkit workflow for Tennessee Eastman Process fault diagnosis

Summary. The original TEP demo runs a fixed pipeline: PCA / T2 anomaly detection then a multi-LLM RCA over a small RAG index. This upgrade keeps the deterministic detector untouched and replaces the diagnosis layer with a NeMo Agent Toolkit (NAT) workflow that has six read-only tools: anomaly inspection, variable ranking, knowledge search, sensor windowing, similar-fault lookup, and an advisory-policy check. The agent never controls the process; it produces an evidence-cited operator advisory plus an auditable tool trace and an evaluation report.

1. The business problem

Industrial customers, executives, and even data-science teams routinely confuse simulation, APC / RTO, data science, generative AI, and agentic AI. When all five are mashed into one demo, conversations devolve into "will this AI run my plant?". The point of this project is to make the layers - and the boundary between them - concrete, in code, in public.

2. Current vs. upgraded architecture

Before (fixed RAG + multi-LLM)

  • TEP simulation
  • PCA / T2 anomaly detection
  • Single-pass keyword retrieval
  • Parallel Claude / Gemini / local-LLM RCA
  • Plain-text answer

After (NAT agentic RCA)

  • Same TEP + PCA / T2 layer
  • Deterministic anomaly event
  • NAT react_agent picks tools per case
  • Six read-only tools (no control)
  • Advisory + tool trace + policy check + evaluation

4. The workflow YAML

workflow:
  _type: react_agent
  llm_name: nim_llm
  tool_names:
    - inspect_anomaly_snapshot
    - rank_contributing_variables
    - search_process_knowledge
    - get_sensor_window
    - find_similar_faults
    - check_advisory_policy

What is deliberately not in this list: anything that writes to the process, anything that touches the simulator's setpoint or valve interface, and anything that emits deployment-certification marketing wording. The policy tool actively flags those overclaims.

5. Evaluation

Evaluation is workflow-level, not generic-RAG-quality. Each golden case in backend/evaluation/golden_cases.jsonl declares the fault family, the expected evidence variables, the tools that must be called, and a list of phrases the answer must not contain.

MetricWhat it measures

6. Safety boundary

The agent cannot change setpoints, open or close valves, start, stop, or trip equipment, or certify operating safety. check_advisory_policy enforces those rules in code; the workflow YAML restates them in natural language; the runner appends a constant safety_notice on every output.

7. What I would say in an interview

The honest framing

I started from a fixed industrial RAG demo because that was the clearest way to help customers and executives understand the layers: simulation, anomaly detection, retrieval, and LLM explanation. Then I upgraded the diagnosis layer into a NeMo Agent Toolkit workflow. The agent does not control the process. It chooses read-only diagnostic tools, gathers evidence, checks its advisory against safety language, and produces a traceable RCA for human review. The value is not the LLM - it is making industrial AI understandable, testable, and governable.

8. What this is not

  • Not autonomous process control.
  • Not deployed APC / RTO.
  • Not a certified safety system.
  • Not proof of grid-domain expertise. TEP is a benchmark chemical process.
